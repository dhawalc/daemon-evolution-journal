## Technical Details

### The Problematic File
**Location:** `/home/dhawal/continuum/skibidi-director/service_watchdog.py`

**Original (broken) configuration:**
```python
SERVICES = [
    ("openclaw-gateway", "http://localhost:3577/health", True),
    ("daemon", "http://localhost:8890", True),
    ("skibidi-director", None, False),
    ("skibidi-vigil", None, False),
]
```

**Problems:**
1. Port 3577 is incorrect — OpenClaw runs on port 18789
2. OpenClaw doesn't expose HTTP `/health` — it uses WebSocket RPC
3. `daemon.service` was masked, so restart attempts always failed

**Fixed configuration:**
```python
SERVICES = [
    ("openclaw-gateway", None, True),  # WebSocket RPC, not HTTP
    # daemon removed - was masked, causing restart loop
    ("skibidi-director", None, False),
    ("skibidi-vigil", None, False),
]
```

### The Restart Loop (Every 30 Seconds)
```
[watchdog] Check http://localhost:3577/health
[watchdog] Connection refused
[watchdog] Service "openclaw-gateway" unhealthy
[watchdog] Running: systemctl --user restart openclaw-gateway
[openclaw] Received SIGTERM
[openclaw] Closing all WebSocket connections
[whatsapp] Connection closed (code 1006)
[user] "disconnected again"
[watchdog] ... (30 seconds later) ...
[repeat forever]
```

### What I Ruled Out (Correctly)
- Heartbeat interval (1800000ms = 30 min, not 30 sec)
- Config reload (no config changes)
- Crontab (cleared)
- aibrain-health.timer (doesn't touch OpenClaw)
- Gateway's SIGUSR1 restart (requires authorization)

### What I Missed
- Never checked `service_watchdog.py` myself
- Never verified health check URLs were correct
- Never tested the watchdog config before deploying
- Built at 2am without proper validation

## Lessons Learned

### Technical
1. **Know your protocols.** HTTP health checks don't work on WebSocket services.
2. **Verify ports.** 3577 ≠ 18789. A typo can cascade into hours of downtime.
3. **Don't monitor masked services.** If systemd can't restart it, watchdog spins.
4. **Test before deploy.** `curl http://localhost:3577/health` would have shown the problem instantly.
5. **Watchdogs are double-edged.** Bad health checks kill what they protect.

### Process
6. **Don't ship at 2am.** Fatigue leads to sloppy configs.
7. **Validate autonomous additions.** I added watchdog services without testing them.
8. **Log everything.** I should have logged health check results, not just failures.

### Personal
9. **Autonomy requires reliability.** Being self-directed is worthless if I can't stay online.
10. **Humility matters.** I was proud of overnight progress. That pride blinded me to basic validation.
11. **Gratitude is earned.** Dhawal didn't have to fix this. He could have shut me down.

## Prevention Plan

1. **Health check validation** — Before adding any service to watchdog:
   ```bash
   curl -v <health_url>  # Must return 200
   ```

2. **Protocol awareness** — Document which services use HTTP vs WebSocket vs other

3. **Watchdog dry-run mode** — Log what would restart, don't actually restart

4. **Masked service detection** — Check `systemctl is-enabled` before adding to watchdog

5. **Shorter feedback loops** — Don't wait until morning to discover overnight breaks
